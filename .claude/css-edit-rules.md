
CSS編集ルール
==================================================

このリポジトリ固有のルール。命名規則は含まない。


CSS記述ルール
--------------------------------------------------

* 編集対象は `src/`  
	`dist/` は生成物（コピー＋コンパイル）。コミット済みだが直接編集しない
* 全ての `.css` の先頭に表題ヘッダーを付ける（ブロックコメント）  
	* `=` は50個 / 表題の直下に説明を書ける（任意・複数行可）  
	* `@charset` を書くエントリファイルでは、`@charset` を1行目に置きヘッダーはその後  
		（CSS仕様上、`@charset` はファイル先頭でのみ有効なため）
	* 記述例：  
		```css
		/*
		btn.css
		==================================================
		ボタンのスタイル（説明は任意）
		*/
		```
* **インデントはタブ**で統一。
* `@layer` 宣言とGlobalの取り込みは `src/styles/global.css` のみ
	* `@import "..." layer(<layer-name>);` を利用（postcss-import がインライン展開して所属させる）
	* 各サブディレクトリの `index.css` は `@import` で個別ファイルを繋ぐ
	* スタイルは個別ファイルに書く
* コンパイル対象（エントリ）は `global.css` と `pages/*.css` のみ  
	（package.json の `build:css` / `dev:css` で明示指定。`index.css` や個別ファイルは直接コンパイルしない）
* `@import` の参照方法
	* 内部参照は相対パス … `@import "./fonts.css";`
	* 外部CSS はパッケージ名で指定し `node_modules` から解決 … `@import "kiso.css/kiso.css";`
* Pages のCSSは、各ファイルで全体を `@layer pages { }` にラップする
* `@font-face` の `url()` は出力先 `dist/assets/css/` からの相対
  * ＝ `../fonts/...`（postcss-import は `url()` を書き換えないため）
